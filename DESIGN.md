## 1. 開発方針

ACDSeeのように、大量の画像を軽快に探し、選び、確認できる軽量画像ビューアを目標にする。

最初のMVPでは、画像編集アプリではなく「高速フォルダビューア」に絞る。重視する体験は、フォルダを開いたら画像一覧がすぐ見え、クリックした画像を大きく表示でき、左右キーで連続確認できること。

優先度は次の順とする。

1. フォルダを選択できる
2. 画像一覧を読み込める
3. サムネイルを表示できる
4. クリックした画像を大きく表示できる
5. 左右キーで前後の画像へ移動できる
6. 大量画像でもUIが固まりにくい
7. 後続Stepでソート、サムネイルサイズ変更、メタ情報、レーティングなどを追加する

Step1では入れないもの。

- 画像編集
- RAW現像
- 顔認識
- AI分類
- タグ管理
- レーティング
- 重複検出
- 印刷
- クラウド同期
- 高度なExif編集
- 複雑なアルバム管理

## 2. 画面レイアウト案

基本はACDSee風の3ペイン構成にする。

```text
+--------------------------------------------------------------------------------+
| ツールバー: フォルダを開く / 戻る / 進む / サムネイルサイズ / ソート             |
+----------------------+------------------------------+--------------------------+
| フォルダツリー       | サムネイル一覧               | 画像プレビュー           |
|                      |                              |                          |
| - ドライブ           | [thumb] [thumb] [thumb]      | 選択画像を大きく表示     |
| - よく使う場所       | [thumb] [thumb] [thumb]      | フィット表示             |
| - フォルダ階層       | [thumb] [thumb] [thumb]      | 前後移動                 |
|                      |                              |                          |
+----------------------+------------------------------+--------------------------+
| ステータスバー: 現在フォルダ / 画像数 / 選択中ファイル / 読み込み状態            |
+--------------------------------------------------------------------------------+
```

各領域の役割。

- フォルダツリー: Windowsエクスプローラー風にフォルダを選ぶ。
- サムネイル一覧: 選択フォルダ内の画像をグリッド表示する。選択中画像は明確にハイライトする。
- 画像プレビュー: クリックした画像を大きく表示する。最初は画面に合わせる表示を基本にする。
- ツールバー: フォルダ選択、戻る/進む、サムネイルサイズ、ソートなどを置く。
- ステータスバー: ファイル名、画像数、読み込み状態などを表示する。

Step1では、見た目を作り込みすぎず、操作の流れを優先する。

## 4. 使用ライブラリ案

第一候補。

- GUI: `PySide6`
- 画像処理: `Pillow`
- テスト: `pytest`
- パッケージ管理: `uv`

理由。

- `PySide6` はデスクトップアプリ向けで、フォルダツリー、リスト、グリッド、ショートカット、スレッド処理が扱いやすい。
- `Pillow` は JPEG / PNG / WebP / BMP / GIF / TIFF など基本形式に対応しやすい。
- `pytest` はロジック部分の検証を小さく始められる。
- `uv` はPython環境と依存関係を軽く管理しやすい。

将来候補。

- `pillow-simd`: 環境が合う場合の画像処理高速化。
- `rawpy`: RAW対応を追加する段階で検討。
- SQLite: レーティング、タグ、履歴などを保存する段階で検討。

## 5. 高速化方針

MVPでは「見えている範囲を速くする」ことを最優先にする。

Step1で入れる最小方針。

- 一覧表示では原寸画像を直接大量に読まない。
- サムネイルは必要になったものから生成する。
- サムネイルのディスクキャッシュを設計に入れておく。
- キャッシュキーは `ファイルパス + 更新日時 + ファイルサイズ` を基本にする。
- プレビュー画像は表示中1枚と前後数枚だけをメモリに持つ。
- 左右キー移動に備えて、次3枚、前1枚程度を先読みする。
- 巨大画像は画面サイズに合わせた縮小済み表示を優先する。

後回しにする高速化。

- 全フォルダの一括スキャン。
- 全画像の事前サムネイル生成。
- 複雑なキャッシュDB。
- GPU最適化。
- RAW現像の高速化。

## 6. Step1で実装する内容

Step1は、ユーザー指定のMVPをそのまま満たす範囲に限定する。

実装対象。

1. フォルダ選択
   - ボタンまたはメニューから任意フォルダを選択できる。
   - 将来のフォルダツリーに備えた構成にする。

2. 画像一覧読み込み
   - 選択フォルダ内の画像ファイルを検出する。
   - 対応形式は `jpg`, `jpeg`, `png`, `webp`, `bmp`, `gif` を基本にする。
   - 自然順または名前順で並べる。

3. サムネイル表示
   - 中央ペインにサムネイルをグリッド表示する。
   - ファイル名を小さく表示する。
   - 選択中のサムネイルをハイライトする。

4. 画像プレビュー
   - サムネイルをクリックしたら右ペインに大きく表示する。
   - ウィンドウサイズに合わせてフィット表示する。

5. キー操作
   - 左キーで前の画像へ移動する。
   - 右キーで次の画像へ移動する。
   - 先頭/末尾ではクラッシュせず、その場に留まる。

6. 最低限のエラー耐性
   - 空フォルダで落ちない。
   - 画像以外のファイルを無視する。
   - 壊れた画像を読んでもアプリ全体が落ちない。

Step1では実装しないもの。

- レーティング、タグ、編集、削除、RAW、Exif詳細、重複検出、スライドショー。
- 高度なDB管理。
- 凝ったアニメーションやテーマ機能。

- 画像フォルダを選択できる。
- JPG / PNG / WebP など主要形式を表示できる。
- サムネイル一覧が表示される。
- サムネイルクリックで大きなプレビューが表示される。
- 左右キーで前後画像へ移動できる。
- 空フォルダで落ちない。
- 画像以外のファイルが混在しても落ちない。

- フォルダ切替後に古い選択状態が残らない。
